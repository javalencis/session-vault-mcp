# Proposal: macos-installer-hardening

## Intent

Improve installer diagnostics and surface clear failure messages to facilitate diagnosing macOS-specific issues without prematurely guessing root causes. Ensure consistency between install docs, installer output, and generated MCP command shape.

## Scope

### In Scope
- Add install-time diagnostics (OS, shell, Node/npm versions, npm prefix, failing step)
- Clarify supported install flows (global npm, npx, linked) in docs and CLI
- Enhance `doctor` command to validate installation method mismatch, not just MCP entry presence
- Add test coverage for packaging and install-surface assumptions

### Out of Scope
- Implementing speculative macOS-specific fixes for installation
- Changing the primary installation method away from npm

## Capabilities

### New Capabilities
- `installer-diagnostics`: Diagnostics-first hardening and validation for session-vault installation flows.

### Modified Capabilities
None

## Approach

Implement a **Diagnostics-first hardening** strategy. Update `scripts/install.sh` to capture and report actionable environment data (OS, shell, Node/npm versions) on failure. Enhance `src/cli/doctor.ts` to verify the actual command execution path matches the installed mode (global vs npx). Ensure documentation aligns perfectly with the actual paths generated by `src/cli/opencode-integration.ts`.

## Affected Areas

| Area | Impact | Description |
|------|--------|-------------|
| `scripts/install.sh` | Modified | Add detailed diagnostics and failing step reporting |
| `README.md` | Modified | Clarify install flows and alignment with MCP command form |
| `src/cli/opencode-integration.ts` | Modified | Ensure command shape is explicit based on install mode |
| `src/cli/doctor.ts` | Modified | Validate command shape matching the chosen install method |
| `test/cli/` | New/Modified | Add coverage for install assumptions and doctor checks |

## Risks

| Risk | Likelihood | Mitigation |
|------|------------|------------|
| Script/package version drift | Medium | Document exact versions and rely on npm registry source of truth |
| `PATH` resolution fragility | High | Explicitly surface path resolution issues in diagnostics |
| Breaking existing installations | Low | Changes are additive diagnostics and validation, not destructive |

## Rollback Plan

Revert changes to `scripts/install.sh`, `src/cli/doctor.ts`, and `src/cli/opencode-integration.ts`. Restore `README.md` to previous state.

## Dependencies

- None

## Success Criteria

- [ ] Installer script provides actionable environment data on failure
- [ ] `doctor` command correctly surfaces install-method mismatches
- [ ] Documentation accurately reflects generated OpenCode MCP commands
- [ ] No new speculative macOS-specific behavior is introduced
